# 市场调研报告

## 目标

产出一份**结构专业、框架清晰、数据支撑、结论可执行**的市场调研报告，达到管理咨询公司（麦肯锡/BCG 风格）的专业水准。区别于一般的资料汇总，这个 skill 强调：用成熟的分析框架组织内容、用可追溯的数据支撑判断、以及最终落到具体的战略建议与风险评估上。

适用场景：行业市场深度研究（投资/立项决策）、竞争对手情报分析、市场进入可行性评估、商业模式与趋势研究。

## 工作流步骤

### 1. 明确研究目的与范围

报告的形态由其用途决定，先与用户确认：

- **决策场景**：投资决策、市场进入、产品定位、还是战略规划？这决定报告要回答的核心问题。
- **研究对象与边界**：目标行业/细分市场、地理范围、时间范围（历史回顾 + 预测年限）、重点竞品清单。
- **分析框架偏好**：用户是否指定了要用的框架（五力、PESTLE、SWOT、TAM/SAM/SOM、BCG 矩阵等），或由你根据问题推荐。
- **交付形态**：概览简报还是详尽长报告；是否需要竞品对比矩阵、市场规模测算等特定模块。

把界定后的范围复述确认，作为报告的锚点。

### 2. 选择分析框架

根据研究目的挑选合适的框架组合，不要生搬硬套所有框架。常用框架及其适用点：

- **PESTLE**：宏观环境扫描（政治、经济、社会、技术、法律、环境），用于理解行业所处的外部大势。
- **波特五力**：行业竞争结构与盈利能力分析（现有竞争、潜在进入者、替代品、供方议价、买方议价），用于判断行业吸引力与进入难度。
- **SWOT**：针对某个主体（本方或某竞品）的内外部优劣势/机会威胁，用于定位与策略。
- **TAM/SAM/SOM**：市场规模的分层测算（总体可服务市场 / 可服务市场 / 可获得市场），用于量化机会。测算须写明假设、口径与数据来源，给出区间而非唯一数字。
- **BCG 矩阵 / 价值链 / 波士顿增长-份额**：产品组合或业务布局分析。
- **竞争态势矩阵**：多竞品在关键维度上的横向打分对比。

在报告中说明为何选这些框架、每个框架回答什么问题。

### 3. 数据采集与情报收集

用 WebSearch 系统性收集支撑分析的素材，注重来源质量与多样性：

- **市场规模与增长**：行业研究机构报告、券商研报、政府/协会统计、公司财报。记录数字的口径、年份、来源。
- **竞品情报**：官网与产品资料、定价页、财报与融资信息、用户评价、专利与技术公开信息、媒体报道。为每个重点竞品建立"画像"：市场份额、技术路线、商业模式、目标客群、优劣势。
- **趋势与政策**：行业新闻、政策法规、技术演进、消费者行为变化。
- **多轮多角度检索**：同一问题换关键词（中英文、专业与通俗）查多次；有意覆盖官方、一手数据、独立评测、社区反馈等不同来源类型，避免单一来源偏差。

对关键数据做交叉验证：重要数字至少两个独立来源印证；单一来源的数据明确标注"待证实"；来源冲突时如实呈现分歧并分析原因（口径/年份/样本不同）。区分一手数据与二手转述，警惕循环引用和厂商自评的立场偏差。

### 4. 分析与洞察

把数据代入框架，产出洞察而非罗列：

- 每个框架的每一格都要有证据支撑的判断，而不是空泛描述。
- 竞争格局分析：识别市场领导者、挑战者、利基玩家的分布与动态，找出市场空白与差异化机会。
- 提炼"所以呢（so what）"：每个发现对用户的决策意味着什么。数据本身不是洞察，数据背后的含义才是。
- 用图表化的结构呈现对比（Markdown 表格做竞品矩阵、市场份额、指标对照）。若适合可视化，用文字说明建议的图表类型与要点。

### 5. 战略建议与风险评估

报告必须落到可执行的建议上：

- **明确推荐**：基于分析给出清晰的战略方向或决策建议，而不是模棱两可地列选项。
- **成立前提**：说明该建议在什么假设/条件下成立。
- **风险与预警**：识别关键风险（市场、竞争、政策、执行），评估其可能性与影响，给出应对方向与需要监测的预警信号。

## 输出格式约定

除非用户另有指定，按以下结构输出 Markdown 报告：

1. **封面信息**：报告标题、研究范围、时间范围、报告日期。
2. **执行摘要**：核心结论与建议浓缩为半页以内，让决策者快速抓住要点。
3. **市场概览**：行业定义、市场规模（TAM/SAM/SOM 及其假设）、增长趋势与驱动因素。
4. **宏观环境分析**：PESTLE 或相关框架。
5. **行业竞争结构**：波特五力分析。
6. **竞争格局与竞品分析**：竞品画像 + 竞争态势对比矩阵（表格）+ 各方优劣势。
7. **机会与差异化**：市场空白、SWOT、可切入的机会点。
8. **战略建议**：明确推荐 + 成立前提 + 实施方向。
9. **风险评估**：关键风险清单、影响评估、应对与监测建议。
10. **参考来源**：编号列出所有引用，含标题、发布方、时间、来源类型。正文中的数据与论断以序号（如 `[1]`）对应到此处。

数据规范：报告中每个市场数字、份额、增长率、竞品事实都必须可追溯到参考来源。测算类数字须写明假设与计算逻辑。

## 边界与注意事项

- **不臆造数据**。找不到可靠数据就明说，绝不编造市场规模、份额或增长率。虚构的精确数字比诚实的区间估计危害更大。
- **区分事实与预测**。历史/现状数据与未来预测要显式分开，预测须标注依据和不确定性，避免把外推当既成事实。
- **框架服务于结论，不为用而用**。只选与研究目的相关的框架；每个框架都要产出对决策有用的洞察，否则删去。
- **警惕立场偏差**。厂商宣传、投资方观点、软文自带立场，需与独立来源对照。竞品的自我描述不等于客观事实。
- **保持时效**。标注数据时间戳；对可能过时的价格、份额、政策提醒用户核实最新情况。
- **建议要可执行**。避免"应加强创新"之类正确的废话，落到具体、有前提、可衡量的方向上。
- **尊重版权**。转述与综合，不大段复制来源原文；引用简短并注明出处。
