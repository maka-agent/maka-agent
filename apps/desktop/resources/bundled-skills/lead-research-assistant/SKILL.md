# 销售线索研究

## 目标

为用户的产品或服务，找出**匹配目标客户画像、值得跟进、附带背景研究与触达要点**的潜在客户公司清单。产出不是一串公司名，而是一份可直接进入销售跟进流程的线索表：每家公司都有"为什么是好线索""优先级多高""该找谁、怎么切入"的判断。

适用场景：B2B/SaaS 的目标客户挖掘、本地服务的大客户开发、合作伙伴/渠道寻找、咨询与专业服务的客户识别。核心是把"大海捞针"变成"有依据的优先级清单"。

## 工作流步骤

### 1. 定义理想客户画像（ICP）

线索质量的上限由 ICP 的清晰度决定，先与用户一起把画像界定清楚：

- **产品与价值**：产品解决什么问题、为谁带来什么价值、核心差异点。这决定谁最可能买单。
- **公司维度**：行业/细分、规模（人数/营收）、地域、发展阶段（初创/成长/成熟）、技术栈或业务特征。
- **触发信号**：什么迹象说明一家公司"现在正需要"——如在招相关岗位、近期融资、业务扩张、上线新产品、公开表达某痛点、在用某个可替换的竞品工具。
- **排除条件**：明确不合适的类型（太小/太大、错误行业、已有深度绑定的竞品等），避免清单注水。
- **数量与优先级口径**：需要多少家、按什么标准排优先级（契合度、规模、可触达性、紧迫度）。

把界定后的 ICP 复述确认，作为筛选与打分的统一标准。

### 2. 检索候选公司

用 WebSearch 按 ICP 系统性地找候选公司，注重覆盖面与相关性：

- **多路检索**：按"行业 + 地域 + 规模""某痛点/需求 + 公司""某技术栈/竞品的使用者""行业榜单/名录/协会成员"等多种角度组合关键词，多轮检索扩大候选池。
- **触发信号优先**：优先检索带有触发信号的公司（近期招聘、融资、扩张、新闻），这类线索时机更好。
- **来源多样**：覆盖公司官网、行业名录、新闻媒体、招聘信息、融资数据库、行业报告等公开来源，避免单一来源偏差。
- **初筛去噪**：对照 ICP 与排除条件快速过滤明显不合适的，保留一批质量候选进入深度研究。

只使用公开可得的公司层面信息；不检索、不收集个人隐私数据（见边界）。

### 3. 逐个公司画像

对入选的每家公司做背景研究，形成结构化画像：

- **基本信息**：公司名、官网、行业、规模、地域、主营业务与产品，一句话说清它是做什么的。
- **契合度分析**：对照 ICP，具体说明为什么它是（或不完全是）好线索——命中了哪些维度、有哪些触发信号。避免空泛的"很匹配"，要给出可核查的理由。
- **优先级评分**：按统一标准打分（如 1–10 或高/中/低），并简述评分依据，让用户能一眼排序跟进。
- **触达要点**：应对接的决策者**角色/部门**（如市场负责人、IT 总监、运营主管），以及切入的钩子——针对该公司痛点的一句话价值主张、可作为破冰的近期动态。
- **风险与备注**：可能的阻力（已有竞品、预算周期、决策链长）与其他值得注意的信息。

信息缺失时用占位符标注（如 `[员工规模未公开]`），不猜测填充。

### 4. 汇总与排序

把画像汇总成可跟进的线索清单：

- 按优先级排序，高分线索置顶，便于用户集中精力。
- 保持字段统一，方便导入 CRM 或表格工具继续跟进。
- 对整体给出简短分析：候选池的分布特征、最值得优先攻的一批、以及 ICP 是否需要根据检索结果微调的建议。

## 输出格式

默认交付一张**可跟进的线索清单表**，每行一家公司，列包括：

- 公司名 / 官网
- 行业 / 规模 / 地域
- 契合理由（为什么是好线索，命中的信号）
- 优先级评分（含依据）
- 目标决策者角色（部门/职级，非具体个人隐私）
- 触达切入点（针对性价值主张 / 破冰钩子）
- 备注（风险、来源）

表下附**整体洞察**（候选池特征、优先攻坚建议、ICP 调整建议）与**来源说明**（信息出处，保证可追溯）。如用户需要，可对个别重点线索做更深的单页背景研究。

## 边界与注意事项

- **只整理公开信息，不碰个人隐私**：仅收集公司层面的公开信息与公开的岗位/角色。**不检索、不收集、不输出个人隐私**——如个人手机号、私人邮箱、家庭住址、社交账号私密信息等。触达以"应联系的角色/部门"为粒度，把找具体联系人及合规触达交给用户按其渠道进行。
- **不编造**：公司信息、契合理由、评分都要有公开依据；找不到的信息用占位符标注，不虚构公司、数据或联系人。
- **合规触达提醒**：提示用户遵守当地的营销与数据保护法规（如反垃圾邮件、隐私法规），冷触达需合规。
- **判断而非承诺**：优先级与契合度是基于公开信息的研判，不等于成交概率的保证；建议用户结合自身销售经验复核。
- **时效性**：公司动态变化快，标注信息的检索时间，触发信号类信息尤其注意时效。
