# SEO 优化审计

## 目标

对一个网站或业务做**覆盖全面、按优先级排序、可直接执行**的 SEO 诊断。产出不是泛泛的"要做好 SEO"，而是一份指出具体问题、说明为什么重要、给出如何修的行动清单——让用户（或其开发/内容团队）能照着一条条落地。

审计范围覆盖 SEO 的主要支柱：技术 SEO、页面级（on-page）优化、关键词与搜索意图、内容质量、结构化数据、外链（off-page）。适用场景：新站上线前体检、排名/流量停滞的诊断、改版前后的对照检查、竞争分析驱动的优化。

## 工作流步骤

### 1. 明确审计对象与目标

先与用户确认背景，让审计聚焦于真正的业务问题：

- **站点与范围**：目标域名/URL；是全站还是重点栏目/页面；技术栈（是否 SPA、CMS 类型）会影响可抓取性判断。
- **业务与受众**：网站的目标（获客/电商/内容/品牌）、目标用户、主营的产品或服务，这决定"该为哪些词排名"。
- **目标市场与语言**：面向的地区/语言，涉及本地 SEO 或多语言（hreflang）时尤其重要。
- **已知痛点**：用户观察到的问题（流量下滑、某些页不收录、排名上不去），作为审计的重点线索。
- **可访问性**：能否抓取页面、是否有 Google Search Console / Analytics 数据可参考。

把范围与目标复述确认，作为审计的锚点。

### 2. 技术 SEO 检查

技术层决定搜索引擎能否顺利抓取、索引页面，是排名的地基。可用 **Bash** 抓取并检查关键文件与响应（如 `curl -sI <url>` 看状态码与响应头、`curl -s <url>/robots.txt`、`curl -s <url>/sitemap.xml`、抓取页面 HTML 检查 meta/标签）。逐项核查：

- **可抓取与可索引**：`robots.txt` 是否误屏蔽重要路径；页面 meta robots / X-Robots-Tag 是否有意外的 `noindex`；`sitemap.xml` 是否存在、是否列出正确 URL。
- **HTTP 与状态码**：是否全站 HTTPS、有无混合内容；关键页返回 200；有无死链（404）、异常 5xx；重定向是否规范（301 而非链式跳转）。
- **规范化**：`canonical` 标签是否正确，是否存在重复内容/多 URL 指向同一页；www/非 www、http/https、带斜杠与否是否统一。
- **站点性能与 Core Web Vitals**：加载速度、LCP/CLS/INP 表现，图片是否压缩、是否用现代格式、是否懒加载。
- **移动友好**：是否响应式、视口设置、移动端可用性（Google 以移动优先索引）。
- **渲染**：若为 SPA/JS 重度站点，核对关键内容是否在初始 HTML 中可见（否则可能不被有效索引）。
- **国际化**：多语言站点的 `hreflang` 是否正确成对。

无法用 Bash 抓取时（如需登录、反爬），说明限制并改用公开信息或让用户提供数据。

### 3. 页面级（On-Page）优化检查

逐页/逐模板核查页面本身传递给搜索引擎的信号：

- **Title 标签**：是否唯一、含目标关键词、长度合适（避免被截断）、对用户有吸引力。
- **Meta description**：是否撰写、是否概括页面并含关键词、能否提升点击率（虽非直接排名因素）。
- **标题层级**：每页一个 H1 且准确概括主题；H2/H3 层级合理、含相关词、组织清晰。
- **URL 结构**：简洁、语义化、含关键词、层级合理，避免冗长参数。
- **图片**：`alt` 文本是否描述性且含相关词、文件名是否语义化、尺寸是否优化。
- **内链**：是否用描述性锚文本把权重传递到重要页面，重要页是否有足够内链指向。
- **关键词落位**：目标词是否自然出现在 title、H1、正文开头与关键位置，无堆砌。

### 4. 关键词与搜索意图分析

用 **WebSearch** 研究目标词的机会与竞争，确保内容对准用户真正在搜的东西：

- **核心与长尾词**：围绕业务梳理主关键词与长尾变体，覆盖用户不同阶段的搜索。
- **搜索意图**：判断每个词背后的意图（信息型/导航型/交易型/商业调查），页面类型与内容应与意图匹配。
- **竞争与机会**：观察当前该词的搜索结果——谁在排名、内容形态如何、是否有精选摘要/图片/视频等 SERP 特征，评估切入难度与机会点。
- **差距分析**：对照现有页面，找出"有需求但无对应页面"的空白，以及"页面存在但未对准意图"的错位。

对搜索量、难度等无法精确获取的指标，给出定性判断并标注需用专业工具（如 GSC、关键词工具）进一步核实。

### 5. 内容与结构化数据检查

- **内容质量**：是否满足搜索意图、深度与实用性是否足够、是否体现经验/专业/权威/可信（E-E-A-T）、是否有过时或单薄（thin content）页面；相似主题是否内容重复自我竞争（关键词蚕食）。
- **内容覆盖**：是否覆盖了目标主题的关键子话题，能否成为该主题的权威页面。
- **结构化数据（Schema）**：是否合理使用 JSON-LD 标记（如 Article、Product、FAQ、Breadcrumb、Organization、LocalBusiness），标记是否有效、与页面内容一致，以争取富媒体搜索结果。

### 6. 外链（Off-Page）评估与建议

- **现状评估**：基于公开信息与 WebSearch 大致了解站点的外链与提及情况、品牌在行业中的可见度。
- **可执行建议**：给出获取高质量外链的方向（内容营销、行业目录、合作/客座、数字公关、修复失效外链等），强调质量优于数量，规避垃圾链接风险。
- **本地/品牌信号**（如适用）：Google Business Profile、本地目录 NAP 一致性、品牌搜索与口碑。

精确的外链数据需专业工具，标注为需进一步核实，不臆造数字。

## 输出格式

交付一份**分优先级的 SEO 审计报告**：

- **执行摘要**：站点 SEO 现状总评、最关键的 3–5 个问题、预期改善方向。
- **按支柱分节的发现**：技术 / On-Page / 关键词与意图 / 内容 / 结构化数据 / 外链，每节列出发现的问题。
- **可执行清单**：核心交付物——每条问题给出 `[优先级 高/中/低] 问题 → 为什么重要 → 如何修复`，按优先级（影响 × 修复成本）排序，让用户能照做。
- **需进一步核实项**：标注哪些结论受限于工具/数据、建议用哪些工具（GSC、Analytics、专业 SEO 工具）验证。
- **来源与检查记录**：抓取的文件/命令与检索来源，保证可追溯。

## 边界与注意事项

- **不编造数据**：搜索量、外链数、排名等无法精确获取的指标，给定性判断并标注需专业工具核实，不虚构具体数字。
- **抓取合规与限制**：用 Bash 抓取时仅访问公开页面、尊重 `robots.txt` 与合理频率；遇到需登录、反爬或抓取失败时说明限制，改用公开信息或请用户提供导出数据（如 GSC 报表）。
- **随算法演进**：SEO 最佳实践随搜索引擎算法变化，对不确定或有争议的做法如实说明，不承诺具体排名结果——排名受众多外部因素影响。
- **优先级导向**：避免罗列一堆同等重要的问题；始终按业务影响排序，帮用户先做性价比最高的改进。
- **落地可执行**：建议要具体到"改哪里、怎么改"，避免"提升内容质量"这类无法执行的空话。
